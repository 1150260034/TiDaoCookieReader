# 天刀助手 Cookie 读取器 - 使用教程

## 简介

这是一个用于提取天刀助手的登录凭证（Cookie）和角色信息的工具，让你不用抓包也能获取周周载愿活动的签到资格。

**当前版本：v1.2.0**

## 你的任务（只需要做这4步）

```
┌─────────────────────────────────┐
│  ① 打开天刀助手                │
│  ② 读取Cookie                  │
│  ③ 选择角色（如有多个）         │
│  ④ 复制全部                    │
└─────────────────────────────────┘
```

## 准备工作

### 1. 安装 MuMu 模拟器

1. 下载 MuMu 模拟器官网安装包
   - 地址：https://mumu.163.com/
   - 选择「MuMu模拟器」下载

2. 安装完成后，启动模拟器

3. **开启 Root 权限**（必须！否则无法读取Cookie）
   - 打开 MuMu 模拟器设置
   - 进入「高级设置」
   - 找到「获取 Root 权限」选项
   - 开启后重启模拟器

### 2. 安装天刀助手

1. 在模拟器中打开浏览器
2. 访问天刀助手官网下载
3. 安装天刀助手 APK

### 3. 安装 Cookie 读取器（TiDaoCookieReader）

1. 将 `app-debug.apk` 复制到模拟器中
2. 直接拖拽 APK 文件到模拟器窗口安装

## 操作流程

### Step 1: 打开天刀助手

1. 点击「① 打开天刀助手」按钮
2. 在天刀助手中**扫码登录 QQ**
3. 在天刀助手中点击「周周载愿」按钮

### Step 2: 读取 Cookie

1. 切换回「天刀Cookie助手」App
2. 点击「② 读取Cookie」按钮
3. 等待几秒，App 会自动读取

### Step 3: 选择角色（如有多个）

如果一个账号有多个角色，会弹出角色选择框：
- 可以选择一个或多个角色
- 勾选后点击「确定」

### Step 4: 复制凭证

1. 点击「④ 复制全部」
2. Cookie 和角色信息已复制到剪贴板

### Step 5: 发给我

把复制的内容**发送给我**，我来完成后续的签到配置。

## 重要提示

**切换账号后**：如果在天刀助手中切换了 QQ 账号，请**重启天刀助手**后再读取 Cookie，否则可能读取到旧账号的信息。

## 常见问题

### Q: App 提示「需要开启 Root」
A: 请在 MuMu 模拟器设置中开启 Root 权限，然后重启模拟器。

### Q: 读取 Cookie 失败
A: 请确保：
1. 在天刀助手中扫码登录了 QQ
2. 在天刀助手中点击过「周周载愿」按钮
3. 然后再点击「② 读取Cookie」

### Q: 切换账号后读取的还是旧账号
A: 这是因为 Cookie 缓存问题。请完全退出天刀助手（不是切换账号），重新打开后再读取。

### Q: 角色选择框没出现
A: 说明该账号只有一个角色，会自动选中。如果有多个角色，第一次读取时会弹出选择框。

### Q: 为什么需要 Root 权限？
A: 因为 Cookie 存储在天刀助手的私有目录，需要 Root 才能读取。

## 配置规范

**重要规则：一个游戏角色 = 一个独立配置**

- 每个角色单独创建配置，不能多个角色混用一个配置
- 同一个QQ账号有多个角色时，每个角色需要单独提取和配置
- 配置ID和名称统一使用角色名，方便识别

示例：
```
账号A (QQ: 123456) 有 角色1、角色2
→ 需要创建 2 个独立配置
→ 配置1: id=角色1, name=角色1, cookie=账号A的cookie, role_params.playername=角色1
→ 配置2: id=角色2, name=角色2, cookie=账号A的cookie, role_params.playername=角色2
```

## 注意事项

1. Cookie 有效期有限，如失效请重新获取
2. Cookie 只用于周周载愿签到，不会用于其他用途
3. 扫码登录时确保是你自己的 QQ 号
4. 切换角色后需要重新读取 Cookie
